Task: Estruturar e Priorizar Requisitos
Usando os requisitos identificados na Etapa anterior, o analista precisa organizar e estruturar os requisitos identificados de acordo (por exemplo, por processos de negócios ou por funções do sistema).
Uma prioridade deve ser identificada pelo cliente para as funcionalidades principais do sistema. Prioridades podem ser declaradas como:
▸ "Alta" - uma funcionalidade que deve ser implementada
▸ “Média” - uma funcionalidade que deve ser implementada
▸ “Baixa” - uma funcionalidade que pode ser implementada
A saída desta etapa é uma lista de requisitos organizados no documento de requisitos.
