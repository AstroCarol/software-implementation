Task: Identificar e Capturar os Requisitos
Tendo em mente os principais conceitos relacionados ao domínio de negócios do cliente, o analista pode iniciar a identificação de requisitos. Nenhuma das situações nos projetos de TI é idêntica. Em alguns casos, a maioria dos requisitos já está identificada num documento. No entanto, na maioria dos casos, os principais requisitos são mencionados apenas oralmente pelo cliente. O analista deve identificar e listar os principais requisitos do sistema a ser construído. Durante esta etapa, o analista não deve começar a detalhar os requisitos identificados. O objetivo principal é obter uma visão abrangente dos requisitos do sistema.
